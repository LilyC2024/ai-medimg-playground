# Limitation Improvements Summary

This extra pass focused on the limitations that were realistically fixable inside the current repository. The most important change was to the Day 3 classical brain-ish mask, which used to collapse badly under the fixed thresholds. I replaced that with an adaptive threshold sweep that scores multiple candidates by continuity, component quality, head-volume ratio, and bone overlap. On the sample CT, the brain-mask voxel count increased from 1,476 to 10,731 and the non-empty slice count increased from 12 to 25, giving the model a much more plausible pseudo-label target.

The second major fix was methodological rather than purely algorithmic. Because the project only has one series, the old Day 4 and Day 5 path evaluated on train fallback. I added buffered intra-series holdout bands so the index now uses 17 train slices, 4 validation slices, 4 test slices, and 4 buffer slices. That still is not patient-level generalization, but it is more honest than measuring on the exact same slices used for optimization. With the refreshed labels and class-balanced loss weighting, the best Day 5 eval Dice improved from 0.2456 to 0.3302.

I also added temperature scaling to reduce the calibration problem in the uncertainty output. The holdout expected calibration error dropped from 0.4526 to 0.1588, and that calibrated temperature is now stored in the checkpoint and used in both the regular inference script and the ONNX deployment runtime. After re-exporting ONNX from the improved checkpoint, the Day 7 deployment Dice against the refreshed pseudo labels improved from 0.2745 to 0.3331, and repeated CLI runs still produced identical mask hashes.

Some hard limits remain. There is still no manual clinical ground truth, there is still only one real CT series, and the scan is still thick-slice data. So this pass does not magically make the project clinically validated. What it does do is make the pipeline meaningfully stronger, more transparent, and less misleading about what its current evidence actually supports.
